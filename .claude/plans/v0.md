## Context

Every Claude Code session starts from zero. Sessions are stored locally as JSONL files but are not searchable or connected. We want to build a pipeline that:
1. Converts session JSONL → searchable markdown (compatible with QMD + Obsidian)
2. Indexes them via QMD for fast retrieval (BM25 + semantic search)
3. Runs automatically via SessionEnd hook — zero manual steps

This is inspired by Artem Zhutov's approach (docs/x-post.md) but built as our own plugin.

## Milestone 0: Schema Validation (derisk) ✅

**Goal:** Prove we can parse a real JSONL and extract meaningful content.

**Tasks:**
1. Copy one real session JSONL to `tests/fixtures/sample-session.jsonl` (truncated to ~50 lines)
2. Write `scripts/parse_session.py` — a standalone Python script that:
   - Reads a JSONL file line by line
   - Classifies each line by `type` field (`user`, `assistant`, `file-history-snapshot`, `progress`, `system`)
   - Extracts from `user` lines: `message.content` (text string or tool_result array)
   - Extracts from `assistant` lines: `message.content` array → filter for `type: "text"` entries only (skip `thinking`, `tool_use` signatures)
   - Extracts tool use summaries from assistant `tool_use` content blocks: `{tool_name}: {brief input summary}`
   - Extracts metadata: `sessionId`, `cwd`, `gitBranch`, `timestamp`, `version`, `permissionMode`
3. Run against the sample fixture — verify output manually

**JSONL Schema (confirmed from real data):**
```
Each line: { type, sessionId, cwd, gitBranch, timestamp, uuid, parentUuid, message: { role, content }, ... }

type="user":    message.content = string (user text) or array with tool_result objects
type="assistant": message.content = array of { type: "text"|"thinking"|"tool_use", ... }
  - text blocks: { type: "text", text: "..." }
  - thinking blocks: { type: "thinking", thinking: "...", signature: "..." } — SKIP
  - tool_use blocks: { type: "tool_use", name: "ToolName", input: {...} } — SUMMARIZE
type="file-history-snapshot": internal bookkeeping — SKIP
type="progress": hook progress events — SKIP
type="system": system messages — SKIP
```

**Verify:** Script runs on real JSONL, outputs clean conversation text.

## Milestone 3: SessionEnd Hook ✅

**Goal:** Wire up automatic export on session close.

**Tasks:**
1. Create `scripts/export_session.sh` — the hook entrypoint:
   ```bash
   #!/bin/bash
   # Reads hook JSON from stdin
   # Calls Python converter
   # Copies to vault
   # Updates QMD index
   # Git push (if configured)
   ```
2. The shell script:
   - Reads stdin JSON → extracts `transcript_path`, `session_id`, `cwd`
   - Calls `python3 scripts/session_to_md.py --input "$transcript_path" --output ~/vault/sessions/`
   - Output filename: `{date}_{project}_{session_id_short}.md` (e.g. `2026-02-04_openclaw_a2f9b374.md`)
   - Runs `qmd update` to re-index
   - Optionally runs `qmd embed` (can be deferred/async since it's slower)
   - Runs `git -C ~/vault push` if the vault is a git repo

3. Register the hook in `~/.claude/settings.json`:
   ```json
   {
     "hooks": {
       "SessionEnd": [{
         "type": "command",
         "command": "/path/to/scripts/export_session.sh"
       }]
     }
   }
   ```

**Verify:** Close a Claude Code session → check `~/vault/sessions/` for new markdown file → `qmd search` finds it.

## Key Design Decisions

1. **Frontmatter includes `source_jsonl` path** — enables tracing from QMD search result → markdown → original JSONL
2. **Tool uses are summarized, not dumped** — keeps markdown searchable without noise
3. **Thinking blocks are skipped** — they're internal reasoning, not useful for search
4. **`qmd update` runs sync, `qmd embed` can run async** — fast hook execution, semantic search catches up
5. **Filename includes date + project + session_id prefix** — human-browsable in Obsidian, no collisions
